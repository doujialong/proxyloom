# ADR-0002：双表示无损节点模型

- 状态：Accepted
- 日期：2026-07-17
- 对应需求：RAW-001 至 RAW-005、PIPE-004、PIPE-008

## 背景

把所有协议压进最小公共结构会丢失未知字段，也无法保证 sing-box 输入仍输出 sing-box 时保持原始语义。相同连接还可能在一个来源中出现多次，连接身份不能兼任列表出现身份。

## 决策

每个已接受快照同时保存四层数据：

1. `RawDocument`：加密保存响应原文和格式检测结果。
2. `RawNode`：保存从文档切出的原生 JSON 值或 URI 原文、提取路径和原始名称。
3. `CanonicalNode`：适配器生成的可查询派生视图；允许不完整，永远不能覆盖原始层。
4. `NodeOccurrence`：来源内跨快照延续的逻辑出现记录。

流水线产生版本化 Patch，不产生新的“真相节点”。同格式渲染从 RawNode 深拷贝，只应用白名单路径上的 Patch；跨格式渲染才使用 CanonicalNode。未知字段永远随 RawNode 保留。

### 指纹

- 已登记协议：适配器输出版本化连接身份投影，再以独立 Fingerprint Key 执行 HMAC-SHA-256。
- 未知结构化协议：对无损有序 JSON AST 删除格式级显示字段后使用 `opaque-json-v1` 编码再计算 HMAC；对象键按 UTF-8 码点排序、数组保持顺序、字符串保持精确解码值、数字保留验证后的原始词法。sing-box 的格式级显示字段仅为顶层 `tag`。该路径不得把数字转换为 IEEE-754 浮点。
- 未知 URI：仅当已登记的容器格式明确声明 URI fragment 不参与连接时排除 fragment，否则对完整原文计算 HMAC。
- 不透明指纹只说明规范化原始结构相同，UI/API 必须使用 `opaque_structural_match`，不得标记为协议语义重复。

所有结构化 JSON 在进入 Raw Node AST 前拒绝重复对象键；不能依赖解析器保留第一个或最后一个值。原始文档仍可按失败保留策略加密留档，但重复键节点不得进入可发布 Snapshot。

Fingerprint 记录 `algorithm`、`projection_version` 和 `key_id`。轮换 Master Key 只重包 Fingerprint Key，不改变摘要。投影升级采用旧、新双写和双读迁移。

### Node Occurrence 关联

同一来源的新快照按以下确定顺序关联：

1. 唯一的同版本 Fingerprint 匹配。
2. 多个相同 Fingerprint 时，先匹配相同提取路径，再按已有 Occurrence 创建时间和 ID 配对。
3. 连接字段变化时，仅在协议、原始名称和格式级位置形成唯一候选时继承。
4. 仍有歧义时创建新 Occurrence，旧记录进入 `absent`，不得猜测。

完全相同且不可区分的多个对象使用稳定的 occurrence 槽位。数量减少时保留最早槽位，数量恢复且仍在 30 天内时优先恢复缺席槽位。该规则保持名称集合稳定，但不声称能识别两个字节相同对象的物理个体。

关联算法版本写入每个快照和构建清单。升级算法不得后台静默重配已有 Name Allocation。

## 后果

- 数据量高于仅存标准化模型，但能重放、审计和适配未来字段。
- 对未知协议可兑现同格式无损，而跨格式转换明确失败。
- 用户修改显示名称不改变已登记格式的连接指纹；修改连接字段通常产生新指纹，但可能由唯一辅助证据继承 Occurrence。
