## 1. 审计范围

本次不重复第一次审计，而是专门复查 D-001 至 D-016 写入正文后产生的二阶影响：

1. 全部保留重复节点与稳定数字后缀是否和格式优先级、过滤、无损语义一致。
2. 节点身份、跨快照匹配、健康结果复用、密钥启动及外部执行器是否可安全实现。
3. 新增行为是否具备验收、容量、恢复、发布和 GitHub 命名闭环。

## 3. 第二轮：实现与安全

| RR2-04 | High | Compose 无法凭描述安全地“自动创建卷外密钥”，覆盖与权限行为未定义。 | 增加一次性 `init-key` 服务/CLI、256 bit 随机性、`0600`、拒绝覆盖和 readiness 失败规则。 | 已修复 |

| RR2-08 | Medium | 镜像内集成 GPL 执行器只有“附许可证”要求，缺少精确版本、修改状态和源码获取方式。 | OPS-005 增加镜像内容和许可证清单发布门槛。 | 已修复 |

## 5. 名称检索证据

2026-07-16 使用 GitHub Repository Search 和公开注册表检查：

- `nodeloom`：GitHub 返回 8 个名称匹配仓库，至少 `xizko39/nodeloom`、`harshhadiyal7/NodeLoom` 为精确同名；`github.com/nodeloom` 已存在。
- `proxyloom`：未发现精确同名 GitHub 仓库或用户；未发现 PyPI、npm 包；Docker Hub namespace 查询结果为 0 个仓库。
- `relayloom`、`subspindle`：同轮检查未发现 GitHub 名称匹配或上述注册表包，可作为备选。

这些结果只能降低混淆风险，不构成商标法律意见；公开发布前仍需终检。
